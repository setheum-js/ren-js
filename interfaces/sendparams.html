Interface SendParams

The parameters for the send function - a common interface that aims to abstract away minting and burning into a single cross-chain transfer function.

Optional confirmations

confirmations: undefined | number
  • Defined in ren-interfaces/build/main/parameters.d.ts:173

For minting, the number of confirmations to wait before submitting the signature to Ethereum. If this number is less than the default, the RenVM transaction is returned when those confirmations have passed, before the signature is available, and will not be submitted to Ethereum.

sendAmount

sendAmount: NumberValue
  • Defined in ren-interfaces/build/main/parameters.d.ts:162

The amount being transferred cross-chain. For sending to an Ethereum address, this is a suggested amount only. The transfer will continue even if a different amount is sent to the Gateway address.

sendTo

sendTo: string
  • Defined in ren-interfaces/build/main/parameters.d.ts:156

The receiving address. If this address is an Ethereum address, then the transfer will be a lock and mint - and will be a burn and release otherwise.
